# 个人作品集网站 — 产品需求文档

## 产品概述

一个展示全栈开发能力的个人作品集网站，以暗色开发者风格和项目深度叙事为核心，面向招聘方和技术合作者。

## 目标用户

- **主要用户 — 招聘方（HR / 技术负责人）**：简历太薄，需要看到实际项目能力。想快速判断技术栈匹配度和项目经验深度。
- **次要用户 — 技术合作者（其他开发者）**：不知道你的技术水平和协作风格。想看你做了什么项目、代码质量、技术品味。
- **边缘用户 — 潜在客户（需要外包/咨询）**：找不到靠谱的全栈开发者。想看项目案例、服务能力和联系方式。

## 界面设计要求

### 整体布局（单页，自上而下）

```
┌─────────────────────────────────┐
│  浮动导航（桌面端顶部固定）        │
├─────────────────────────────────┤
│  Hero + 个人简介                  │  全屏高度
│  姓名 / 职称 / 头像 / 一句话简介    │
│  简介文字直接跟在 Hero 下方        │
├─────────────────────────────────┤
│  项目展示（核心区）                │
│  项目卡片 × N，每张包含：          │
│  截图 + 项目描述 + 技术标签 + 链接  │
├─────────────────────────────────┤
│  技能展示                        │
│  紧凑标签云形式，分类：            │
│  后端 / 前端 / 工具与平台          │
├─────────────────────────────────┤
│  联系方式                        │
│  邮箱 + 社交链接（GitHub 等）      │
└─────────────────────────────────┘
  右下角：返回顶部悬浮按钮
```

### 各区块设计要点

- **Hero + 个人简介**：暗色背景，居中布局。姓名 + 职称标题（大字号），头像圆形，下方一段个人简介。双语切换按钮放于右上角。
- **项目展示**：页面核心区，每个项目为独立卡片。卡片结构：截图（上方）→ 项目名称 + 描述（中间）→ 技术标签行（下方）→ 源码/演示链接。点击卡片可展开详情弹窗，包含项目背景、技术决策、成果数据（P1）。
- **技能展示**：紧凑标签云，按"后端 / 前端 / 工具与平台"三个分类排列，每个技能以彩色 tag 呈现。
- **联系方式**：页面底部，邮箱 + GitHub 等社交图标链接，简洁收尾。

### 导航与交互

- **桌面端**：顶部浮动导航条，包含 Logo（姓名）+ 区块锚点链接 + 语言切换按钮。滚动时高亮当前所在区块。
- **移动端**：汉堡菜单折叠导航项，语言切换保留在外层。
- **返回顶部**：右下角固定悬浮按钮，滚动超过一屏后出现，点击平滑滚动回顶部。
- **hover 微交互**：项目卡片 hover 轻微上浮 + 阴影增强；技能 tag hover 色彩饱和度提升。

### 设计风格

- 暗色开发者风格，参考 Tamal Sen（tamalsen.dev）
- 色彩：暗色基底 + 小面积彩色细节点缀（分隔线、hover 状态、技能 tag）
- 字体：系统字体栈优先，中文可配合无衬线字体

|------|------|------|------|

## 非功能性需求

| 维度 | 要求 |
|------|------|
| 运行平台 | 现代浏览器（Chrome / Firefox / Safari / Edge 最近 2 个大版本） |
| 性能 | Lighthouse ≥ 90，LCP < 2.5s，CLS < 0.1 |
| 安全性 | HTTPS（Vercel 自带），CSP header 基础配置 |
| 交付形式 | Vercel 部署 + 自定义域名，Git push 自动部署 |

## 代码风格和架构模式

### 目录结构

```
src/
├── components/          # 通用 UI 组件（Navbar, BackToTop, ProjectCard 等）
├── sections/            # 页面区块，每个区块独立目录
│   ├── Hero/
│   ├── Projects/
│   ├── Skills/
│   └── Contact/
├── hooks/               # 自定义 hooks（useScrollSpy, useI18n 等）
├── i18n/                # 中英文翻译 JSON 字典
│   ├── zh.json
│   └── en.json
├── types/               # TypeScript 类型定义
├── styles/              # Tailwind 配置 + 全局样式
└── utils/               # 工具函数
```

### 代码风格

- 变量/函数：camelCase；组件：PascalCase；文件名与默认导出组件同名
- TypeScript 严格模式开启
- 默认不加注释，只在 WHY 不明显时写一行短注释
- 组件优先使用函数组件 + Hooks

### 架构模式

- 单页 SPA，纯前端静态站点，无后端服务
- 组件化架构：每个区块为独立目录，内含该区块专用组件 + 样式 + 类型
- 数据流：静态数据定义在 `src/data/` 目录（个人信息、项目列表、技能列表），组件直接 import
- 国际化：`react-i18next`，语言偏好存入 localStorage，默认检测浏览器语言

## 约束条件和边界场景

### 明确不做（V1 范围外）

- 博客 / 内容管理后台（CMS）
- 亮色模式切换（V1 仅暗色主题）
- 联系表单提交功能（联系方式仅展示邮箱 + 社交链接）
- 用户认证 / 登录
- PWA / 离线支持 / Service Worker
- 项目筛选 / 搜索功能
- CLI / 终端交互元素

### 边界场景

- 不支持 IE11 及更早版本浏览器
- 不考虑项目数 > 20 的极端分页场景
- 不做完整 WCAG 无障碍审计（语义化 HTML + alt 文本会做）

### 技术限制

- 纯前端静态站点，无后端服务
- 部署依赖 Vercel 平台

# 个人作品集网站 — 产品需求文档

## 产品概述

一个展示全栈开发能力的个人作品集网站，以暖色纸白风格和项目深度叙事为核心，面向招聘方和技术合作者。

## 目标用户

- **主要用户 — 招聘方（HR / 技术负责人）**：简历太薄，需要看到实际项目能力。想快速判断技术栈匹配度和项目经验深度。
- **次要用户 — 技术合作者（其他开发者）**：不知道你的技术水平和协作风格。想看你做了什么项目、代码质量、技术品味。
- **边缘用户 — 潜在客户（需要外包/咨询）**：找不到靠谱的全栈开发者。想看项目案例、服务能力和联系方式。

## 界面设计要求

### 整体布局（单页，自上而下）

```
┌─────────────────────────────────┐
│  浮动导航（桌面端顶部固定）        │
├─────────────────────────────────┤
│  Hero + 个人简介                  │  全屏高度
│  姓名 / 职称 / 头像 / 一句话简介    │
│  向下滚动提示                      │
├─────────────────────────────────┤
│  项目展示（核心区）                │
│  全屏轮播图，中间大图 + 两侧窥视    │
│  鼠标悬停边缘 → 跟随按钮自动切换    │
│  点击 → 详情弹窗（背景/方案/成果）  │
├─────────────────────────────────┤
│  技能展示                        │
│  紧凑标签云形式，分类：            │
│  后端 / 前端 / 工具与平台          │
├─────────────────────────────────┤
│  联系方式                        │
│  邮箱 + 社交链接（GitHub 等）      │
└─────────────────────────────────┘
  右下角：返回顶部悬浮按钮
```

### 各区块设计要点

- **Hero + 个人简介**：暖色背景，居中布局。姓名 + 职称标题（大字号），头像圆形，下方一段个人简介，底部向下滚动提示箭头。
- **项目展示**：页面核心区。全屏宽度轮播图，中间当前项目占主体（约 64%），左右两侧可见相邻项目边缘（各 20% 半透明）。鼠标移至左/右边缘 → 出现跟随鼠标的"上一个/下一个"浮动按钮，停留 0.8 秒自动切换。点击中间大图 → 详情弹窗。
- **技能展示**：紧凑标签云，按"后端 / 前端 / 工具与平台"三个分类排列，每个技能以彩色 tag 呈现。
- **联系方式**：页面底部，邮箱 + GitHub 等社交图标链接，简洁收尾。

### 导航与交互

- **桌面端**：顶部浮动导航条，包含 Logo（姓名）+ 区块锚点链接 + 语言切换按钮。滚动时高亮当前所在区块。
- **移动端**：汉堡菜单折叠导航项，语言切换保留在外层。
- **返回顶部**：右下角固定悬浮按钮，滚动超过一屏后出现，点击平滑滚动回顶部。
- **轮播交互**：鼠标悬停边缘 → 跟随按钮出现 → 0.8s 自动切换；点击边缘立即切换；键盘 ← → 切换。
- **开发模式编辑**：`import.meta.env.DEV` 限定，生产环境不可访问。点击姓名编辑个人信息，项目标题旁编辑按钮管理项目。

### 设计风格

- 暖色纸白风格，参考 Squarespace
- 色彩：暖纸白基底（#f5f4f0）+ 微暖灰卡片（#ebeae5）+ 蓝色强调（#0066cc）
- 字体：标题 JetBrains Mono，正文系统字体栈 + Noto Sans SC
- 项目截图以浅色背景衬托，不喧宾夺主

## 代码风格和架构模式

### 目录结构

```
src/
├── components/          # 通用 UI（Navbar/BackToTop/ProjectCarousel/SkillTag/LanguageSwitch/Skeleton/SEO/ProfileEditor/ProjectEditor）
├── sections/            # 页面区块（Hero/Projects/Skills/Contact），每区块独立目录
├── hooks/               # useScrollSpy/useEditableProfile/useEditableProjects/ProfileContext
├── i18n/                # zh.json + en.json 翻译字典 + index.ts 初始化
├── data/                # 静态数据：profile.ts/projects.ts/skills.ts
├── types/               # Profile/Project/Skill 接口
└── styles/              # globals.css（Tailwind 指令 + CSS 变量）
```

### 架构模式

- 单页 SPA，纯前端静态站点，无后端服务
- 组件化架构：每个区块为独立目录
- 数据持久化：编辑数据存 localStorage，无编辑时使用 `src/data/` 静态默认值
- 国际化：`react-i18next`，语言偏好 localStorage 持久化，默认中文
- 编辑功能：`import.meta.env.DEV` 限定，生产构建自动移除

## 约束条件和边界场景

### 明确不做（V1 范围外）

- 博客 / 内容管理后台（CMS）
- 联系表单提交功能（联系方式仅展示邮箱 + 社交链接）
- 用户认证 / 登录
- PWA / 离线支持 / Service Worker
- 项目筛选 / 搜索功能
- CLI / 终端交互元素

### 边界场景

- 不支持 IE11 及更早版本浏览器
- 不考虑项目数 > 20 的极端分页场景
- 不做完整 WCAG 无障碍审计（语义化 HTML + alt 文本会做）

### 技术限制

- 纯前端静态站点，无后端服务
- 部署依赖 Vercel 平台

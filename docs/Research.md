# 个人作品集网站 — 需求研究

## 我的原始想法

- **要解决的问题**：需要一个线上门面来聚合展示个人品牌、技术能力和项目经验，方便潜在雇主/合作方/客户了解我。
- **给谁用**：招聘方、技术合作者、潜在客户。
- **为什么现有方案不够好**：目前没有个人网站，GitHub 等线上存在不够聚合和直观。

### 核心板块

| 板块 | 内容 |
|------|------|
| 个人介绍 | 姓名、头像、个人简介、联系方式 |
| 技能展示 | 掌握的技术栈 |
| 项目展示 | 项目截图、描述、链接 |

### 非功能需求

- 响应式设计，手机和电脑都能正常显示

## AI 的分析

### 技术可行性

个人作品集网站是 web 开发中成熟度最高的品类，技术难度低。核心工作在于内容组织和视觉设计，而非技术攻坚。

- **难度**：低。静态站点生成器（Astro/Next.js/Gatsby）或纯 HTML/CSS/JS 均可实现。
- **开发周期**：内容齐备的前提下，MVP 可在 1-2 周内完成。
- **部署运维**：Vercel/Netlify/GitHub Pages 免费托管，零运维成本。

### 潜在风险

1. **内容瓶颈**：项目截图、描述文案的准备往往比代码开发更耗时。建议在开发启动前先整理好所有项目素材。
2. **视觉设计短板**：后端偏强、前端偏弱的情况下，容易做出"能用但不好看"的效果。通过参考已验证的设计模式（暗色主题 + 色彩点缀）可有效规避。
3. **响应式坑点**：移动端导航、图片适配、触摸交互需要专门测试，不能只靠框架默认行为。
4. **长期维护**：作品集网站的最大风险是"做完就忘"——内容过时后反而产生负面印象。建议每季度更新一次项目列表。

### 替代方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **从零开发**（当前方案） | 完全可控、展示开发能力、可扩展 | 开发成本较高 |
| **GitHub Pages + Jekyll/Hexo** | 零成本、Markdown 驱动、专注内容 | 定制化受限、模板感强 |
| **Framer/Webflow 低代码** | 设计品质高、快速上线 | 月费、迁移成本、不能证明开发能力 |
| **直接使用开源模板**（如 bchiang7/v4） | 最快上线、社区验证过 | 差异化弱、仍需一定定制 |

> 推荐当前方案（从零开发），因为作品集本身就是一个项目——它能证明你能独立完成一个完整的 web 产品。

## 竞品调研

### 主要参考：Tamal Sen（tamalsen.dev）

**身份**：全栈/前端 & Flutter 开发者  
**设计公式**：暗色基底 + 彩色微交互 + 浮动导航 + 单页叙事 + 返回顶部

| 维度 | 分析 |
|------|------|
| **色彩** | 深色背景为主，彩色细节点缀（分隔线、hover 状态、tag 标签），视觉层次清晰 |
| **布局** | 单页滚动，所有内容按区块排列（Hero → About → Skills → Projects → Contact） |
| **导航** | 浮动/固定导航，滚动时保持可见，指示当前所在区块 |
| **交互** | 返回顶部按钮、hover 微交互、滚动驱动的色彩变化 |
| **整体感受** | 专业但不花哨，开发者气质浓厚 |

**借鉴点**：
- 暗色主题 + 色彩点缀的配色策略——用低成本实现高品质视觉
- 浮动导航 + 返回顶部——两个小功能显著提升单页浏览体验
- 单页布局适合内容量中等的场景（3-5 个区块）

### 其他参考案例

| 产品 | 优点 | 缺点 | 借鉴点 |
|------|------|------|--------|
| **Brittany Chiang** (brittanychiang.com) | 极简卡片式、开源（v4）、信息层级清晰 | 差异化弱、Gatsby 技术栈稍旧 | 项目卡片布局、sticky 侧边栏导航 |
| **Arek "System Status: ONLINE"** | 终端/CLI 风格、后端开发者气质极强、创意独特 | 非技术访客可能不理解 | CLI 交互元素可作为 Hero 区域的亮点 |
| **abdulbasit 终端式作品集** | Next.js 15 + Tailwind、实时 API 端点、命令行交互 | 内容承载量有限 | API 端点证明后端能力、Markdown 驱动内容 |
| **Bento Grid 作品集** (v0.app) | 2025-2026 最热布局、视觉冲击力强、信息分区清晰 | 需要较多内容块支撑、移动端适配成本高 | 网格化分区思路可用于项目展示板块 |
| **lizuju 极简玻璃态** | 原生 JS 零构建工具、中英双语、GitHub Pages 部署 | 设计感偏弱、功能有限 | 双语支持方案、GitHub Pages 部署思路 |

### 设计趋势总结（2025-2026）

1. **暗色/亮色模式切换**已成为标配
2. **Bento Grid** 便当盒布局是当前最热趋势
3. **微交互**（滚动触发动画、hover 状态）替代大动画成为主流
4. **移动优先**——70%+ 流量来自手机，响应式不是可选项
5. **AI 辅助开发**（Vibe Coding）使视觉品质门槛降低，差异化转向内容深度
6. **性能指标**：Lighthouse 95+、LCP < 1.5s、CLS 为零

## 结论

### 这个项目值得做吗？

**值得。** 个人作品集是开发者的线上身份名片，投入产出比高。且作品集本身就是一个完整的前端项目——它能证明你有独立交付 web 产品的能力。

### 核心差异化

你作为**后端偏强的开发者**，作品集的差异化不在于前端炫技，而在于：

1. **项目深度 > 视觉复杂度**：每个项目展示的不是截图，而是"解决了什么问题、你怎么决策的、结果如何"
2. **暗色开发者风格**（Tamal Sen 路线）：用低成本实现高品质视觉，重点放在内容叙事
3. **可选亮点**：嵌入实时 API 端点（如 `/api/status`），用后端能力证明自己

### 设计方向确认

以 **Tamal Sen 风格为基准**：
- 暗色基底 + 彩色细节点缀
- 单页布局（Hero → 个人介绍 → 技能 → 项目 → 联系方式）
- 浮动导航 + 返回顶部
- 移动端响应式
